# DeepSeek Technical Evolution

## DeepSeek-V1: The Foundation
* **Standard Dense & MoE:** Implemented a Top-K Mixture-of-Experts (MoE) system where a router assigns tokens to specific sub-networks. Each token is processed by a gating network that selects the most relevant experts. This keeps the computational cost constant while expanding the model's total knowledge base.
* **Domain Specialization:** Introduced specialized checkpoints for coding and mathematics. These were refined using Supervised Fine-Tuning (SFT) on logical datasets, ensuring the model prioritizes structural correctness over creative linguistic patterns.

---

## DeepSeek-V2: The Efficiency Breakthrough
* **MLA (Multi-Head Latent Attention):** MLA uses low-rank joint compression to store Key-Value information in a compact latent vector, compressing KV cache. MLA fixes this by using a "compress-then-decompress" strategy based on Low-Rank Adaptation. It reduces the memory footprint of long-context conversations by 93%, enabling context windows on 128K to 1M+ tokens.
  - Latent Vector Compression: Instead of storing large embedding for separate rows in Key and Value cache, MLA compresses them into a smaller embedding via a linear projection into a lower-dimensional space using a learned matrix of smaller rank.
  - Up-Projection: During the actual "thinking" phase (the attention calculation), the model uses a learned matrix to "up-project" or inflate that tiny latent vector back into the full Key and Value matrices needed for math.
  - Catch: Positional information (knowing that word 1 comes before word 2) is hard to compress. If we compress it, the "coordinates" get blurry. The fix is to use decoupled Rotary Positional Embeddings (RoPE): To keep track of word order, MLA separates the positional information from the content information. It only applies "rotation" (RoPE) to a small portion of the vector, which further saves computation. So we keep the contenct vector compressed, and a position vector, that are combined during decompression.
  - In technical terms, it's better than Multi-Query Attention and Multi-Head attention. MLA provides the efficiency of MQA with the complex logic of Multi-Head Attention. In standard MQA, we have one Key and one Value head that all Query heads must share. In MLA, instead of a single "Head," you store a single latent vector (the compression vector) for each token position. The "efficiency" of MQA comes from reducing Memory Bandwidth Bottlenecks, but maintains the intelligence and precision of Multi-Head Attention (where every head has its own specialized focus).
  - Also increases runtime because bandwidth is the efficiency bottleneck in inference, and this reduces the time moving memory, despite increasing the computation time on compression/decompression.
* **Fine-Grained Expert Segmentation:** An evolution of the MoE architecture. Instead of a few large experts (8 experts and top=k=1 or 2), V2 splits them into hundreds of "micro-experts" (180 experts, top-k=16). It also utilizes "Shared Experts" (cloned) that handle universal language patterns. This increases expert picking precision, i.e. it allows for hyper-specialization. Micro-experts focus on niche knowledge (e.g., specific API documentation), while the Shared Expert prevents redundant learning of basic grammar.

## DeepSeek-V3: Scaling & Hardware Optimization
* **FP8 Mixed-Precision Training:** Optimized for H100/H800 GPU clusters. Training uses 8-bit floating point (E4M3 for forward, E5M2 for backward) with dynamic scaling factors to maintain numerical stability. It doubles the training throughput and halves the VRAM requirement compared to BF16, allowing for larger models without additional compute. Also introduces fine-grained quantization, per tile or per terson.
* **Multi-Token Prediction (MTP):** during Training, model outputs several tokens simultaneously (the main head sits at the end of the trunk and predicts token $i+1$, while the auxiliary MTP heads are smaller, independent "mini-models" (or towers) that fork off from trunk and predict $i+2, i+3$, etc..), creating a much denser learning signal. During Inference,  by default, predicts one token at a time to maintain maximum accuracy, but it can switch to Speculative Decoding mode, to output multiple tokens in one go and verifies them instantly.
  - Parallel Prediction Towers: Instead of having a single output head, the architecture includes multiple additional "prediction towers".
  - Simultaneous Guessing: While the main model predicts the very next token ($n+1$), these secondary towers use the hidden states from previous layers to simultaneously predict tokens further in the future, such as $n+2$, $n+3$, and so on.
  - Auxiliary Training Objective: These future predictions carry their own training loss, meaning the model is penalized if its "guesses" about the distant future are incorrect. 
  - Shared Representation: All prediction heads share the same underlying transformer body, forcing the entire network to develop representations that are useful for long-term planning. 
  - Makes sense because of global coherence: forcing the model to care about what comes after the next word, it develops a stronger internal "plan" and a more global understanding of sentence structure and logic. Leads to improved Reasoning: Training with MTP encourages the model to look for long-range dependencies in data, helpful for maintaining logic in complex coding and mathematical proofs.
  - Inference Speed: This setup allows for Speculative Decoding, where the model can generate several tokens in a single forward pass instead of just one, significantly increasing generation speed.
* **Auxiliary-Loss-Free Load Balancing:** A routing improvement for MoE that uses a bias-addition mechanism to ensure experts are used equally without needing a separate loss function that might conflict with the primary language task.
* **[DualPipe](https://github.com/deepseek-ai/dualpipe):** A specialized pipeline parallelism algorithm that overlaps the computation of the "forward" and "backward" passes with the communication between GPU nodes. Splits input in 2 micro-batches launched in two directions (first to last GPU and vice-versa). This eliminates "pipeline bubbles" (idle GPU time), ensuring maximum utilization of massive GPU clusters.
* **Reasoning (CoT) Capabilities:** Integration of Chain-of-Thought reasoning developed through Large-Scale Reinforcement Learning (RL), teaching the model to use a `<think>` block to verify steps before providing an answer.

## DeepSeek-V4: The Intelligence Revolution
* **Engram Architecture:** A decoupled factual memory system that offloads static knowledge (facts, documentation) into a high-speed "Engram" retrieval bank, keeping the main transformer layers dedicated to reasoning. This stops the model from "forgetting" facts in long contexts and allows reasoning to happen without the weight of massive factual lookups in every layer.
* **mHC (Manifold-Constrained Hyper-Connections):** Advanced layer connectivity using manifold-aware projections to link distant layers. This ensures that the gradient (learning signal) stays crisp even in models with 100+ layers.
* **U-Shaped Allocation:** A strategic parameter distribution that allocates more parameters to the "input" (understanding) and "output" (generation) phases, while using highly efficient, dense reasoning blocks in the middle. It optimizes the "thought process" by spending compute where it matters most: understanding the prompt and synthesizing the final answer.
